What is OpenTelemetry?
Think of OpenTelemetry as a universal instrumentation toolkit that helps you understand what's happening inside your application.
It's like having black box recorders in your airplane (your Spring Boot app) that capture critical telemetry data.

OpenTelemetry focuses on three main types of data:
1)Traces - Following a request's journey
Like tracking a package through delivery: enters system → gets processed by Service A
→ calls Database → calls Service B → returns response
Each step is a "span" with timing and metadata

2)Metrics - Numerical measurements over time
Request rates, error counts, response times, CPU usage
Like a car's dashboard showing speed, RPM, fuel level

3)Logs - Timestamped event records
Traditional log messages with context
Now enriched with trace IDs to connect them to specific requests

1)Traces
Think of ordering food at a restaurant:
Trace = Your entire dining experience (enter → order → eat → pay → leave)
Span = Each step in that experience (just the ordering part, just the eating part)


@RestController
public class LearningController {

    // Spring gives you this automatically - it's your tool to create traces
    @Autowired
    private Tracer tracer;

    @GetMapping("/learn")
    public String learnTracing() {

        // 🔴 STEP 1: Create a Span
        // REAL MEANING: "I want to start timing and tracking a specific piece of work"
        // Think of this as: You pull out a stopwatch and label it "doing-some-work"
        Span myWorkSpan = tracer.spanBuilder("doing-some-work")
            .startSpan();  // Start the stopwatch NOW

        // 🔴 STEP 2: Make this span the current one
        // REAL MEANING: "Everything from now until I close this scope belongs to this span"
        // Think of this as: You start the stopwatch and focus ONLY on this task
        try (Scope scope = myWorkSpan.makeCurrent()) {

            // 🔴 STEP 3: Add useful information to the span
            // REAL MEANING: "I want to remember these details about this work"
            // Think of this as: Writing notes on your stopwatch about what you're timing
            myWorkSpan.setAttribute("user.id", "123");
            myWorkSpan.setAttribute("work.type", "learning");

            // 🔴 STEP 4: Do the actual work
            // REAL MEANING: This is the code you're actually trying to measure
            Thread.sleep(100);  // Simulate doing something that takes time

            return "I learned tracing!";

        } catch (Exception e) {
            // 🔴 STEP 5: Handle errors
            // REAL MEANING: "Something went wrong - record what happened"
            myWorkSpan.recordException(e);  // Take a picture of the error
            myWorkSpan.setStatus(StatusCode.ERROR);  // Mark this work as failed
            throw e;

        } finally {
            // 🔴 STEP 6: End the span
            // REAL MEANING: "Stop the stopwatch - this work is done"
            // CRITICAL: If you forget this, the trace will think this work is STILL RUNNING
            myWorkSpan.end();}}}

        @RestController
        @RequestMapping("/api/shop")
        public class ShopMetricsController {
            @Autowired
            private MeterRegistry meterRegistry;
            // Different types of metrics for different purposes
            private final Counter orderCounter; //integer
            private final Timer orderTimer;  //time
            private final AtomicLong activeCartsGauge; //Long
            private final DistributionSummary cartValueSummary;

            public ShopMetricsController(MeterRegistry meterRegistry) {
                // 1. COUNTER - How many orders?
                this.orderCounter = Counter.builder("shop.orders.total")
                    .description("Total number of orders placed")
                    .register(meterRegistry);

                // 2. TIMER - How long do orders take?
                this.orderTimer = Timer.builder("shop.order.duration")
                    .description("Time to process an order")
                    .register(meterRegistry);

                // 3. GAUGE - How many active carts?
                this.activeCartsGauge = new AtomicLong(0);
                meterRegistry.gauge("shop.carts.active", activeCartsGauge);

                // 4. DISTRIBUTION - Cart values
                this.cartValueSummary = DistributionSummary.builder("shop.cart.value")
                    .description("Distribution of cart values")
                    .baseUnit("USD")
                    .register(meterRegistry);
            }

            @PostMapping("/checkout")
            public CheckoutResult checkout(@RequestBody Cart cart) {

                // Record cart value before processing
                cartValueSummary.record(cart.getTotalValue());

                // Cart becomes active (gauge up)
                activeCartsGauge.incrementAndGet();

                // Time the entire checkout process
                CheckoutResult result = orderTimer.record(() -> {
                    // Simulate checkout process
                    processPayment(cart);
                    updateInventory(cart);
                    sendConfirmation(cart);
                    return new CheckoutResult("SUCCESS", cart.getId());
                });

                // Cart no longer active (gauge down)
                activeCartsGauge.decrementAndGet();
                // Count this order
                orderCounter.increment();
                // Also count by payment method
                meterRegistry.counter("shop.orders.by.payment",
                    "method", cart.getPaymentMethod()
                ).increment();
                return result;}

            private void processPayment(Cart cart) {
                // Track payment processing time separately
                Timer.Sample paymentSample = Timer.start(meterRegistry);
                try {
                    // Call payment gateway
                    Thread.sleep(100);
                } finally {
                    paymentSample.stop(meterRegistry.timer("shop.payment.time"));}}}

## ✅ The Complete Flow:
## 📋 Step-by-Step:
### 1. Your Spring App (application.properties)**
```properties
otel.exporter.otlp.endpoint=http://localhost:4317
otel.service.name=product-service
↓ Your app sends traces to `localhost:4317`
### **2. Docker Port Mapping**
```yaml
signoz-query-service:
  ports:
    - "4317:4317"  # Forwards localhost:4317 to container
↓ Docker forwards to container
### **3. signoz-query-service receives traces**
```yaml
signoz-query-service:
  environment:
    - CLICKHOUSE_HOST=signoz-clickhouse  # Uses service name
```
↓ Query service saves to ClickHouse
### **4. signoz-clickhouse stores data**
```yaml
signoz-clickhouse
  volumes:
    - clickhouse-data:/var/lib/clickhouse  # Persistent storage
```
↓ Data saved in database
### **5. You view traces**
```yaml
signoz-frontend:
  ports:
    - "3301:3301"  # Web UI
↓ Open browser: `http://localhost:3301`
↓ Frontend asks query-service for data
↓ Query-service reads from ClickHouse
↓ You see traces!
## 🎯 **Visual Flow:**
Your Spring App → localhost:4317 → Docker → query-service → clickhouse → frontend → You
     (sends)        (port map)     (forward)  (process)    (store)    (display)   (view)
**That's the full path from your app to your browser!** 🚀
